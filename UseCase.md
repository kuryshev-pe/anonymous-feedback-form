# Сценарий заполнения формы обратной связи

## Общее описание
Форма обратной связи представляет собой основной компонент веб-приложения, который автоматически отображается при запуске frontend-страницы. Она предназначена для сбора информации от пользователей и предоставляет интерактивный интерфейс для ввода данных.

## Сценарий взаимодействия пользователя

### 1. Запуск приложения
- При открытии веб-страницы автоматически отображается форма обратной связи
- Форма занимает центральное положение на странице
- Пользователь сразу видит поля для заполнения
- Страница загружается с индикатором загрузки (если требуется)

### 2. Ввод данных
Пользователь может:
- Выбрать отдел из выпадающего списка (необязательно)
- Выбрать категорию обращения (обязательно):
  - Предложение по улучшению
  - Проблема в работе
  - Вопрос по зарплате
  - Корпоративная культура
  - Технические вопросы
  - Другое
- Ввести email (опционально, для обратной связи)
- Ввести текст сообщения (минимум 10 символов)

### 3. Валидация данных
- Система проверяет корректность введенных данных:
  - Email должен соответствовать формату (если указан)
  - Категория должна быть выбрана
  - Сообщение должно содержать минимум 10 символов
- При обнаружении ошибок отображаются сообщения об ошибках
- Поля с обязательными данными выделяются

### 4. Отправка формы
- Пользователь нажимает кнопку "Отправить"
- Отображается индикатор загрузки (круговой спиннер)
- Данные передаются на backend-сервер через API
- После успешной отправки отображается уведомление об успехе
- При ошибке отображается сообщение об ошибке

### 5. Завершение взаимодействия
- После успешной отправки форма сбрасывается
- Пользователь получает подтверждение отправки
- Форма остается доступной для повторного использования
- При необходимости пользователь может указать email для обратной связи

## Технические особенности

### Интерфейс
- Адаптивный дизайн, работающий на различных устройствах
- Центрированное расположение формы
- Визуальная обратная связь при вводе данных
- Индикаторы загрузки и статуса операций

### Валидация
- Клиентская валидация данных перед отправкой
- Проверка формата email
- Обязательные поля для заполнения
- Минимальная длина текстового сообщения

### Обратная связь
- Уведомления об успехе/ошибках через Snackbar
- Индикатор загрузки при отправке данных
- Автоматическое сброс формы после успешной отправки

## Пользовательский опыт

### Простота использования
- Интерфейс интуитивно понятен
- Минимальное количество шагов для отправки обратной связи
- Быстрая обработка отправки формы
- Отображение статуса операции (загрузка, успех, ошибка)

### Анонимность
- Форма может использоваться анонимно
- При указании email пользователь получает возможность обратной связи
- Ясное объяснение анонимности в интерфейсе

## Потенциальные сценарии использования

1. **Анонимное обращение** - пользователь не указывает email
2. **Обратная связь с ответом** - пользователь указывает email для получения ответа
3. **Сбор предложений по улучшению** - выбор категории "Предложение по улучшению"
4. **Сообщение о проблеме** - выбор категории "Проблема в работе"
